)    GPIOPinFunCfg(G##eShortPin, ulPinConfig)

Configure the alternate function of a GPIO pin. This function configures the pin mux that selects the peripheral function associated with a particular GPIO pin. Only one peripheral function at a time can be associated with a GPIO pin, and each peripheral function should only be associated with a single GPIO pin at a time (despite the fact that many of them can be associated with more than one GPIO pin).

Parameters
[in]eShortPinSpecified port and pin.
[in]ulPinConfigis the pin configuration value, specified as only one of the GPIO_P??_??? values.
Returns
None.

Returns
None.

Definition at line 956 of file xgpio.h.

#define xGPIOSPinIntClear (   eShortPin)    xGPIOPinIntClear(G##eShortPin)

Clear the interrupt for the specified pin.

Parameters
[in]eShortPinSpecified port and pin.
Returns
None.
Note
Because there is a write buffer in the Cortex-M0 processor, it may take several clock cycles before the interrupt source is actually cleared. Therefore, it is recommended that the interrupt source be cleared early in the interrupt handler (as opposed to the very last action) to avoid returning from the interrupt handler before the interrupt source i actually cleared. Failure to do so may result in the interrupt handler being immediately reentered (because the interrupt controller still see the interrupt source asserted).

Because there is a write buffer in the Cortex-M0 processor, it may take several clock cycles before the interrupt source is actually cleared.
