#define xSPIConfigSet (   ulBase,
  ulBitRate,
  ulConfig 
)    SPICfg(lBase, ulBitRate, ulConfig)

Configures the SPI (synchronous serial interface) This function configures the synchronous serial interface. It sets the SPI protocol, mode of operation, bit rate, and data width and the first bit.

Parameters
[in]ulBasespecifies the SPI module base address. This value can be one of the following value:
  • xSPI0_BASE
[in]ulBitRatespecifies the clock rate. This parameter defines the bit rate for the SPI. This bit rate must satisfy the following clock ratio criteria:
  • 0 != ulBitRate (master mode)
[in]ulConfigis the required configuration of the SPI. The ulConfig parameter is the logical OR of several different values, many of which are grouped into sets where only one can be chosen.

The Protocol of SPI can be one of the following values:

  • xSPI_MOTO_FORMAT_MODE_0
  • xSPI_MOTO_FORMAT_MODE_1
  • xSPI_MOTO_FORMAT_MODE_2
  • xSPI_MOTO_FORMAT_MODE_3

The SPI module can operate as a master or slave; The Mode of SPI can be one of the following values:

  • xSPI_MODE_MASTER
  • xSPI_MODE_SLAVE

The width of the data transfers can be a value between 1 and 32, inclusive. For LPC17xx, this parameter can be one of the following value:

  • xSPI_DATA_WIDTH8
  • xSPI_DATA_WIDTH9
  • xSPI_DATA_WIDTH10
  • xSPI_DATA_WIDTH11
  • xSPI_DATA_WIDTH12
  • xSPI_DATA_WIDTH13
  • xSPI_DATA_WIDTH14
  • xSPI_DATA_WIDTH15
  • xSPI_DATA_WIDTH16

The first bit of the data transfers, can be one of the following values:

  • xSPI_MSB_FIRST
  • xSPI_LSB_FIRST
Returns
None.

Note
For LPC17xx, There is no single Enable/Disable control bit in SPI registers group, so, xSPIEnable try to turn on SPI module power. xSPIDisable try to turn off SPI module power. as result, you MUST to reconfigure SPI module after call xSPIDisable.
